Having trouble connecting to your database in the Wagon native app? We're here to help.

  1. Are you able to connect to the "Demo Postgres Database"?

    Yes: Great, Wagon is working. Move on to step 2.

    No: Check common error messages.

  2. Are you able to connect to the same database in another tool?

    Yes: Ensure you are still able to connect in the other tool and double check your connection information for typos. Check common error messages.

    No: Have you ever connected in another tool?

    • Yes: Confirm with your database administrator that the database is up.
    • No: Check the first time setup instructions.

Common Error messages

  1. ConnectTimeout 10s

    This error message means Wagon stopped trying to connect after 10 seconds. There are a few possible causes:

    1. If you're trying to connect to the Demo Postgres Database, a local network issue (e.g. a firewall) is blocking connections to "testdata.wagonhq.com" on port 5432. Ask your network administrator to allow connections to "testdata.wagonhq.com" on port 5432. You can also retry on another network.
    2. If you're trying to connect to Amazon Redshift or Amazon RDS, you may need to configure your security group to allow connections from your current IP address.
    3. If you're connecting to SQL Server, ensure TCP is enabled.

  2. ERROR 0: Data source name not found and no default driver specified

    Click in the menu: Help -> Install ODBC Drivers and then retry

  3. realpath: does not exist (No such file or directory)

    Quit Wagon, reopen, and try to connect again.

  4. Incorrect SSH private key or password. Double check your SSH credentials.

    Double check your SSH credentials, and if your SSH key has a password, make sure you entered it.

  5. Please check your SSH private key.

    You've given Wagon a file that is not an SSH private key.

  6. Error 2003 (MySQL)

    Either your MySQL Server is not running, or your network is blocking connections to the hostname and port. More info.

SQL for Newbs students

  1. ERROR 0: Data source name not found and no default driver specified

    Click in the menu: Help -> Install ODBC Drivers and then retry.

  2. ConnectTimeout 10s or ERROR 2003: Can't connect to MySQL server on 'udemydb.wagonhq.com'

    A network issue (e.g. a firewall) is blocking connections to "udemydb.wagonhq.com" on port 3306. Ask your network administrator to allow connections to "udemy.wagonhq.com" on port 3306, or retry on your home network.

First time setup

Wagon connects to your database directly from your laptop. Wagon's servers do not have access to your database.
